# Seller and packaging notes

This file is intentionally **practical**, not hype. Pair it with the HTML playbooks for tone and layout.

## What you are selling

Pick one primary offer shape:

1. **Template / toolkit** — AutoTag configuration + playbook bundle + sample outputs.
2. **Managed runs** — you operate scans on a schedule, deliver CSV + narrative summary.
3. **Implementation sprint** — integrate AutoTag outputs into Notion, Airtable, or n8n.

## Proof assets (build from real runs)

- Before/after directory maps (from Phase 1 outputs).
- CSV column highlights: business value, integration flags.
- One anonymized client-style case study (even if it is your own archive).

## Pricing ladder (example structure)

- **Starter:** one-off scan + CSV + 30-minute walkthrough.
- **Pro:** recurring scans + prioritized queue + Slack/Email delivery.
- **Enterprise:** custom categorization model + retention policy + SLAs (only if you will honor them).

## Channels

- Gumroad: fastest for digital bundles.
- Codester / dev marketplaces: emphasize technical depth and folder maps.
- Services: use playbooks as leave-behind, not as legal contract.

## Truth anchor

Revenue copy in HTML must not contradict limits of the tool (AutoTag inventories and scores; it does not replace legal, financial, or compliance review unless you scope that separately).
